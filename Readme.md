# rdp_ext
Экспериментальный проект расширения RDP virtual channel для обмена прикладными сообщениями между клиентом и сервером.

Основная идея проекта — сделать собственный микро-протокол поверх RDP-канала и получить управляемый транспорт для обмена данными.
Главный фокус на на базовой инфраструктуре: протокол, сериализация, логика состояний, отправка и приём сообщений, кроссплатформенные адаптеры к mstsc, wts, freerdp и xrdp.

**DISCLAIMER**: Проект не рассматривается как прикладное расширение для передачи собственных данных поверх уже существующей RDP-сессии. Это просто прототип, сделанный для экспериментов с новыми фишками стандартов и подготовки к собеседованию.

## Особенности реализации
- Минимальная платформозависимость. Платформозависимый код локализован в нескольких файлах
- Разделение протокола, логики, транспорта и рантайма
- Интерфейсы в виде концептов. Экспериментальная фича, попробовал новый стандарт. Упрощают написание и юнит тестирование, эффективнее в runtime чем интерфейсы с виртуальными методами, но замедляют компиляцию и создают неудобства при подключении классов, завязанных на концепты. 

## Известные проблемы
- Не работает xrdp транспорт (apps/xrdp_server). Сообщения уходят без ошибок, но не доходят до клиента
- Не реализована shutdown логика
- Не реализована логика ping
- Нет защиты от повторной инициализации плагина
- Утечки памяти в платформенно зависимых transport, нет нормального освобождения ресурсов
- Не аккуратная работа с this в asio
- Документирование исходников отсутствует (кроме Readme)
- Мало unit тестов
