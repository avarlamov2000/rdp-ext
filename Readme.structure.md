# Структура проекта

### Основные каталоги
- `deps/` - зависимости
- `src/` - исходный код проекта
- `src/apps/` - приложения (wts, xrdp) и динамические библиотеки (mstsc, freerdp)
- `src/core/` - ядро системы (протокол, телеметрия, логирование) 
- `src/logic/` - логика приложения (машина состояний и контроллер)
- `src/runtime/` - рантайм (управляет временем жизни)
- `src/transport/` - транспорт (платфомозависимый код)
- `tests/` - тесты

### Основные модули
- `core` - ядро системы (протокол, телеметрия, логирование), состоит из:
- `core/logging` - подсистема логирования: LogSink и Logger на основе plog
- `core/protocol` - реализация микро-протокола: Message, MessageFactory, Serializer на основе zpp_bits
- `core/telemetry` - сбор телеметрии: CpuSampler, TimerControl
- `logic` - логика приложения (машина состояний и контроллер), состоит из:
- `logic/client` - логика клиента: Controller с использованием контрактов-концептов и ClientStateMachine
- `logic/common` - общие данные логики: состояния, события, эффекты
- `logic/server` - логика сервера: Controller с использованием контрактов-концептов и ServerStateMachine
- `runtime` - рантайм (управляет временем жизни сервера и клиента), а также Executor (сначала был написан на потоке с очередью задач, затем переделан на boost::asio)
- `transport` - транспорт (абстракция над API wts, mstsc, xrdp, freerdp). Самая не доделанная часть, т.к. было много экспериментов из-за не соответствия документации и реального состояния дел
